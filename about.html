Experiments with py5 (by a rubyist)

Here I will document my py5 experiences

Warning my python is very rusty

But I hope that my experience with pyprocessing, processing.py and as developer of ruby-processing projects may give useful insights.

Pyprocessing is a pure python implementation of processing, based on pyglet (so includes numpy), but performance is a bit wanting. Processing.py is similar to JRubyArt using jython to connect to java-processing, and can be run from processing ide. JRubyArt uses jruby instead of jython to connect to java-processing (but no longer relies on vanilla-processing). Although py5 uses some processing java code, it is a more sophisticated animal than its predecessors. Key points are that you should think python first, so for example it does not support PVector, for matrix and vector operations you should use numpy (its fast and powerful).
